See also:

Steps

  1. Using Oracle E-Business Suite, choose System Administrator > Security > User > Define
  2. Find user. If using Employee ID, left pad with zeroes to 11 characters or prefix with %.
  3. To updated an email address, go to the Office Details tab.
  4. To change the proxy, click on the flex field to the right of Latest Start Date, then select from there.
    Note: Access to the WFVC Storefront (WFC_WFCC_TRAINING) requires permission from Katharine Partridge.
  5. Check address against email footer or Teamworks and correct as necessary.
  6. Save and exit.

The official PMT documentation for creating a new store has some spaghetti code that I have removed here. Most of the steps below are high-level and can be expanded for more detail by clicking on them.

Steps

    • In Oracle R12, choose System Administrator > Security > Responsibility > Define
    • Responsibility Name: WFC Store Name.
    • Application: iStore
    • Responsibility Key: WFC_STORE_NAME (no spaces)
    • Available From: Oracle Applications
    • Data Group: Name: "Standard"
      Data Group: Application: "iStore"
    • Menu: "iStore Customer UI Menu Root"
    • Menu Exclusions tab: Ignore, leave blank, whatever. Updated later.
    • Leave blank:
      • Description
      • Web Host Name
      • Web Agent Name
      • Request Group: Name
      • Request Group: Application
      • Menu Exclusions tab: Name
      • Menu Exclusions tab: Description
      • Excluded Items tab
      • Securing Attributes tab
    • Save and close
    • System Administrator > Profile > System
    • Click on Responsibility, then key in Responsibility Name (not Key)
    • Click on Profile. Key "WFC: iStore Store Identifier" and Find.
    • If logo then Responsibility = Y else N
    • Save and close
    • iStore Administrator > iStore Merchant > Catalog > Sections > Hierarchy
    • Select Root, then click Create Section
    • Section Name: Store Name (max 120)
    • Section Code: STORE_NAME (no spaces)
    • Parent: Confirm Root, else search glass to find
    • Status: Published
    • Start Date: Today unless otherwise specified
    • Leave blank and unchecked everything else:
      • Short Description
      • Long Description
      • Keywords
      • End Date
      • Featured Section
      • Apply status to all subsections
      • Enable Product Comparison
    • Click Apply.
    • iStore Administrator > iStore Merchant > Sites > Summary > Summary
    • Create a Site, then Go
      • Site Name: Store name. This is what shows up in the Storefront list.
      • Site Code (max 240, no spaces): Responsibility Key
      • Description: Storefront blurb. This is what shows up in the Storefront list.
      • Start Date: today
      • Default Language: American English
      • Default Currency: USD
      • Default Walk-in Price List: Search and find WFC Storefront Price List
      • Catalog Root Section: Catalog Root Section created above
      • Check: Restrict customer access
      • Leave blank: End Date.
      • Unchecked:
        • Allow un-registered users to browse the site
        • Allow customers to check item availability
        • Enable Threshold for Payment Types
        • Enable Reporting
        • Enable Traffic Filter
    • Click Apply.
    • Click Shipping, then Add Shipping Methods.
      • FedEx 2-Day (2 business days by 10:30 AM)
      • FedEx Ground (3-5 days)
      • FedEx Priority Overnight (1 business day by 10:30 AM)
      • FedEx Saturday
      • MAC MAIL / STANDARD (5-8 days)
      • USPS (5-8 business days)
    • Click Responsibility.
    • Remove IBE_CUSTOMER row. (Or don't. This never sticks and it gets end-dated later.)
    • Add Responsibility. Search by name to find the new store's responsibility.
    • Click Select and Apply.
    • Store Link
      Production http://cppra01a0100.wellsfargo.com:8080/OA_HTML/ibeCAcpSSOReg.jsp
      UAT http://cupra01a0100.wellsfargo.com:8080/OA_HTML/ibeCAcpSSOReg.jsp
      SIT http://eus-storefront-sit.wellsfargo.com:8080/OA_HTML/ibeCAcpSSOReg.jsp
      DEV https://eus-storefront-dev.wellsfargo.com:4523/OA_HTML/ibeCAcpSSOReg.jsp
    • Register as a user of an existing company.
    • Registry ID: 1082
    • First Name: Store Name, no spaces
    • Last Name: WF
    • Email: storename@wellsfargo.com (ficticious)
    • User Name: Proxy user's name, no spaces
    • Password: WFstore01, then confirm it
    • Leave blank or unchecked
      • Middle Name
      • Business Phone Number
      • Personal Phone Number
      • Fax Number
      • Please send me e-mail about special offers and products
    • Submit. Ignore "Choose Your Store: Warning"
    • Repeat for all states if appropriate
    • In R12, CRM HTML Administrator > Setup > Users > Registration > Pending Approvals
    • Click the link under User Name. Do not check Select and then Accept. If you do, Oracle assigns a new account number, which Storefront will not recognize.
    • Click Next in the User Details screen
    • In Account Assignment, click next to 1000 in Account Number, then Next
    • Comments: Approved or something
    • Click Accept
    • Repeat for other proxy users
    • System Administrator > Security > User > Define
    • Click User Name, then Flashlight
    • Key new store proxy into Find then click Find
    • Click the proxy in the User Name section, then OK
    • Description: iStore Proxy User ELID-PROXY_NAME Note: This string of text matters. The case-sensitive "iStore Proxy..." is how the list proxies is generated, and the string past "ELID-" is how it's listed (no spaces).
    • If IBE_CUSTOMER is listed—it probably will be—enter the current date into Effective Dates: To field; otherwise, move on.
    • For any of the following not listed, add them:
      • The new store
      • iSupport Site: Business User
      • WFC General Store
      • WFC Advisor
      • WFC Consumer Credit Solutions
      • WFC ISA Store
      • WFC Regional Bank L&D

      • All States Proxy if appropriate
    • Save
    • System Administrator > Application > Flexfield > Key > Values
    • Click Value Set, key wfc_system_ids in Name, then click Find
    • For each new proxy user:
      • Click somewhere in the Value column, then click New
      • Key the new proxy name in Value, then hit Tab. TV field is popped
      • Key iStore Proxy User in Description
    • Save
    • iStore Administrator > iStore Merchant > Catalog > Sections > Hierarchy
    • On the line with the new store, click the Create Section icon.
      • Section Name: Home
      • Parent Section: verify; locate and fix if wrong
      • Status: Published
      • Start date: Today or when scheduled
    • Leave blank/unchecked:
      • Section Code
      • Short Description
      • Long Description
      • Keywords
      • Featured Section
      • Apply status to all subsections
      • End Date
      • Enable Product Comparisons
    • Click Apply
    • Click Content
    • Search the Section Additional Information field for WFC Store Home Page
    • Click Apply.
    • Templates > Layout: Fixed Go. Apply
    • Templates > Display Template Section Contains Products Only, Go
    • Radio Button: Product multi-select with drill down.
    • Click Apply.
  10. Refer to PMT 28856 – Creating and updating Storefront Home Pages to add images or other "content".
    1. iStore Administration > iStore Merchant > Catalog > Sections > Hierarchy
    2. ◦On the line with the new store, click the Create Section icon.
      • Section Name: Store or Catalog Section name if one of those; "Products" otherwise. Every store must have a Products section.
      • Long description: Blank if not specified. 4k characters

      • If Product list: Default
      • If Nav section with both products and navigational subsections: Nope. Can't be done with the current setup. Confirmed 2016-10-12 working around the International store. You can add a Products section, but the items in it will be invisible.
    6. Click Apply.
    1. iStore Administration > Catalog > Sections > Hierarchy

      • Order by item number: segment1 in the Order by Clause field
    4. Apply
  13. After the tasks above Add proxy SUPERPREVIEW to the new store. (System Administrator > Security > User > Define)
  14. If L&D, add LEARNING_P iStore proxy.
  15. Update iStore proxy user ADMIN to add new store

Calling this a Configuration Report a bit of a misnomer. The task is a matter of refreshing the Access database used for Configurations Pricing. It is refreshed about every two weeks, typically on payday Friday. After refreshing the database, you should generate and update the new SharePoint Item List.

Documentation: Configuration Report R12.docx

See also: SharePoint Item List

Steps

  1. In Oracle SQL Developer, run the indicated query and export each Excel workbook to the filename in Access Data Load:
    Script Export File
    R12 PTO Inventory with LOB.sql R12 PTO Inventory with LOB.xlsx
    PTO POC Models.sql PTO POC models.xlsx
    Config Bill of Materials.sql Bill of Materials.xlsx
    Feature.sql Feature.xlsx
    CZ_RULES.sql CZ Rules.xlsx
  2. Open S:/iStore/Configuration/Configuration Models/Access DB and Configuration SQL.
  3. Copy the Access database R12 Configuration Database.accdb to the Access Database Backups folder as YYYY-MM-DD - R12 Configuration Database.accdb.
  4. Open R12 Configuration Database.accdb.
  5. Empty the tables:
    • Bill of Materials
    • CZ Rules
    • Feature
    • PTO POC models
    • R12 PTO Inventory with LOB in catalog
    Note: There is a macro in the database, Clear Tables, that will quickly empty all five tables.
  6. In Access, under External Data > Saved Imports, run:
    • Import-R12 PTO Inventory with LOB
    • Import-Feature
      Note: This import generates some errors, but you can ignore them. You can safely delete the generated table 'Export Worksheet$'_ImportErrors.
    • Import-CZ Rules
    • Import-Bill of Materials
    • Import-PTO POC models

Documentation: PMT 021679 – Updating the SharePoint Item List

See also: Configuration Report

Steps

  1. Open the Access file R12 Configuration Database.accdb.
  2. In Access, under External Data > Saved Exports, run Export-Sharepoint Item List Step B.
  3. In Access, under External Data > Saved Exports, run Export-Sharepoint Item List Step C.
  4. In Oracle SQL Developer, run Item Numbers used in Configuration R12.sql.
  5. Export the file to an Excel file in S:/iStore/Configuration/Curriculum/ITEM_LIST with a filename indicating the current date.
  6. Open the new workbook as well as Sharepoint Item List Step B.xlsx and Sharepoint Item List Step C.xlsx in the same directory.
  7. Move and combine the data from Sharepoint Item List Step B and Sharepoint Item List Step C into a tab in the new workbook. Remove duplicates and sort. Label the tab All Items.
  8. Label the tab with the big query Attachments.
  9. Format as necessary for readability. Delete the SQL tab.
  10. Update the price list announcement and attach the workbook on the Storefront Onboarding SharePoint site.

EUS Storefront – Bad Email Address

(User Name), the error you encountered when trying to access EUS Storefront was caused by an email address mismatch between the Storefront and ePro applications. I updated your record. Please try again. You should now have access to Storefront. This request has been closed; however, if you still cannot access Storefront, please notify me as soon as possible. I will reopen the request and investigate.

EUS Storefront – Missing Account

(User Name), the error you encountered when trying to access EUS Storefront was caused because you did not yet have an account in the underlying database. I have created that record and updated it with the appropriate information. Please try again. You should now have access to Storefront. This request has been closed; however, if you cannot still access Storefront, please notify me as soon as possible. I will reopen the request and investigate.

Incorrect Mailing Address

(User Name), your Storefront record had not been updated with your current location. I updated your record. Please try again; you should now be access the items for your state. This request has been closed. However, if you cannot still access your state’s item, please notify me as soon as possible; I will reopen it and investigate.

Specific Store Access

(User Name), I updated your Storefront record to grant you access to the specific store. You should now have the access requested. This request has been closed; however, if you do not have the system access requested, please notify me as soon as possible. I will reopen the request and investigate.

Top query only for data validation as needed.

ACCEPT ModelUserName CHAR PROMPT 'Enter a User Name:  '

SELECT 
    FU.USER_NAME,
    FU.DESCRIPTION,
    FU.EMAIL_ADDRESS,
    FU.USER_ID,
    RESP.USER_ID,
    RESP.RESPONSIBILITY_ID,
    RLNAME.RESPONSIBILITY_ID,
    RLNAME.RESPONSIBILITY_NAME
FROM 
    APPLSYS.FND_USER FU 
    INNER JOIN 
        APPS.FND_USER_RESP_GROUPS_DIRECT RESP 
        ON FU.USER_ID = RESP.USER_ID
    INNER JOIN
        APPS.FND_RESPONSIBILITY_TL RLNAME
        ON RLNAME.RESPONSIBILITY_ID = RESP.RESPONSIBILITY_ID
WHERE 
    FU.USER_NAME= '&&ModelUserName';

********************************
********************************  */

ACCEPT ModelUserName CHAR PROMPT 'Enter an E-mail Address:  '

SELECT 
    FU.USER_NAME,
    RLNAME.RESPONSIBILITY_NAME,
    FU.END_DATE
FROM 
    APPLSYS.FND_USER FU 
        INNER JOIN 
            APPS.FND_USER_RESP_GROUPS_DIRECT RESP 
            ON FU.USER_ID = RESP.USER_ID
        INNER JOIN
            APPS.FND_RESPONSIBILITY_TL RLNAME
            ON RLNAME.RESPONSIBILITY_ID = RESP.RESPONSIBILITY_ID
WHERE 
    lower(FU.EMAIL_ADDRESS) = lower('&&ModelUserName')
    AND
    (   FU.END_DATE IS NULL 
     OR
        FU.END_DATE > CURRENT_DATE
    )
ORDER BY
    RLNAME.RESPONSIBILITY_NAME;
/* Removing the substitution variable from memory so that the query can be ran repeatedly. */
UNDEFINE ModelUserName

    SELECT email_address, count(email_address)
    FROM per_all_people_f pap
    WHERE 1=1
